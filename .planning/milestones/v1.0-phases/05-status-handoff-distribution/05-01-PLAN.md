    **Create `Diligent/diligent/helpers/time_utils.py`:**
    Three pure functions, no I/O imports:
    - `parse_since(since_str: Optional[str], default_days: int) -> date`: Accepts None (uses default_days), "Nd" relative format, or ISO date string "YYYY-MM-DD". Returns a `date` object representing the cutoff.
    - `is_recent(entry_date_str: str, cutoff: date) -> bool`: Parses first 10 chars of entry_date_str as ISO date, returns True if >= cutoff. Returns False on empty string, None, or parse failure.
    - `relative_time_str(days_ago: int, ref_date: Optional[date] = None) -> str`: If days_ago <= 14, returns "{N}d ago" (or "today" for 0). If > 14, returns the ISO date string computed from ref_date (defaults to today).

    **Section 1 - Deal Header:**
    Read DEAL.md and STATE.md. Format per locked decision:
    - Post-LOI: `"{deal_code} -- {target_common_name} ({target_legal_name})\nStage: {deal_stage} | LOI: {loi_date} | {days} days in diligence"`
    - Pre-LOI: `"{deal_code} -- {target_common_name} ({target_legal_name})\nStage: {deal_stage} | {days} days tracking"`
    - Days-in-diligence counter: `(date.today() - date.fromisoformat(deal.loi_date[:10])).days` for post-LOI; `(date.today() - date.fromisoformat(state.created[:10])).days` for pre-LOI fallback.

    **Section 2 - Workstreams:**
    Read WORKSTREAMS.md, TRUTH.md, QUESTIONS.md, ARTIFACTS.md. For each workstream:
    - Count facts where fact.workstream == ws.name
    - Count open questions where q.workstream == ws.name and q.status == "open"
    - Count artifacts where a.workstream == ws.name
    - Count stale artifacts for that workstream (from compute_staleness result)
    - Format: `"  {ws.name:<20} {fact_count} facts, {question_count} questions, {artifact_count} artifacts, {stale_count} stale"`
    - Cap at 5 workstreams in non-verbose mode with "  and N more..." line.

    **Section 3 - Stale Artifacts:**
    Use `compute_staleness` from reconcile_anchors.py (lazy import) to get authoritative stale artifacts. This ensures consistency with `diligent reconcile`.
    For each stale artifact: `"  {path}: {len(value_changed) + len(source_superseded)} facts changed, {max_days_stale}d since refresh"`
    - max_days_stale: max of days_stale across all StaleFactInfo entries
    - Cap at 5 in non-verbose with "  and N more..." line.

    **Section 4 - Open Questions:**
    Filter questions where status == "open". For each:
    - Origin tag: `[gate]` if question.context is not None else `[manual]`
    - Truncate question text to 60 chars with "..." if longer
    - Format: `"  {q.id} {origin_tag} {q.workstream:<12} {truncated_text}"`
    - Cap at 5 in non-verbose with "  and N more..." line.

    **Section 5 - Recent Activity:**
    Derive from timestamps across state files, 14-day window. Collect events:
    - Facts: use fact.date, verb "set fact '{key}'"
    - Sources: use src.date_received, verb "ingested {src.id}"
    - Artifacts registered: use a.registered, verb "registered {a.path}"
    - Artifacts refreshed: use a.last_refreshed (only if != registered), verb "refreshed {a.path}"
    - Questions: use q.date_raised, verb "raised {q.id}"
    - Questions answered: use q.date_answered (if not None), verb "answered {q.id}"
    Sort descending by date. Use relative_time_str for display.
    - Format: `"  {relative_time:<8} {verb_text}"`
    - Cap at 5 in non-verbose with "  and N more..." line.

    **Summary line:**
    Count "items needing attention" = stale artifact count + open question count + flagged fact count.
    Print: `"\n{N} items need attention"` (or "No items need attention" when 0).

    **JSON mode:** Build the full data dict and call `json.dumps(data, indent=2, default=str)` via click.echo. The data dict has keys: deal, workstreams, stale_artifacts, open_questions, recent_activity, attention_count.
