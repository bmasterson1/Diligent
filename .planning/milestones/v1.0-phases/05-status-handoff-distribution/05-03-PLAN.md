    ```
    # dd:{domain}

    {One-sentence description for runtime matching.}

    ## When to use
    - {Trigger condition 1}
    - {Trigger condition 2}
    ...

    ## Commands

    ### {subcommand 1}
    ```bash
    {{DILIGENT_PATH}} {command} {args} {flags}
    ```
    {Brief description of behavior.}

    ...

    ## Rules
    - {Rule 1}
    - {Rule 2}
    ...

    ## Common workflows
    ### {Workflow name}
    {Step-by-step sequence of commands for a common task.}
    ```

    **dd_truth.md:**
    - Description: Manage validated facts in the deal's TRUTH.md file.
    - When to use: Recording a data point/metric/finding, checking/verifying a fact, viewing value history, flagging for review.
    - Commands: truth set (with --source, --workstream, --anchor, --confirm, --computed-by, --notes), truth get, truth list (--workstream, --stale, --json), truth trace, truth flag (--reason, --clear).
    - Rules: --source is required on every truth set. Values are stored as quoted strings. Verification gate fires on value changes beyond tolerance; use --confirm to override. Flagged facts are advisory. Never invent values.
    - Workflows: "Record a new fact" (truth set), "Investigate a discrepancy" (truth trace, truth flag), "Review stale facts" (truth list --stale).

    **dd_sources.md:**
    - Description: Register and manage source documents in SOURCES.md.
    - When to use: New document received, checking source history, comparing document versions, updating source metadata.
    - Commands: ingest (with --supersedes, --workstream, --parties, --notes), sources list (--json), sources show, sources diff (--verbose, --json).
    - Rules: Source IDs are auto-generated (DEAL_CODE-NNN). Supersedes chain tracks document lineage. Relative paths stored for cross-platform sync. Source documents are read-only.
    - Workflows: "Ingest a new source" (ingest), "Compare two versions" (sources diff), "Check source history" (sources show).

    **dd_artifacts.md:**
    - Description: Track deliverables and check staleness against truth facts.
    - When to use: Registering a deliverable, checking what needs refresh, running staleness check, marking something as updated.
    - Commands: artifact register (--references, --workstream, --confirm, --notes), artifact list (--stale, --json), artifact refresh, reconcile (--workstream, --strict, --all, --verbose, --json).
    - Rules: Artifacts reference truth keys via --references. Staleness means a referenced fact changed after last refresh. reconcile is the authoritative staleness engine. --strict exits non-zero on staleness for scripted checks.
    - Workflows: "Register a deliverable" (artifact register), "Check what needs refresh" (reconcile), "Mark deliverable updated" (artifact refresh).

    **dd_questions.md:**
    - Description: Track open questions and manage the answer lifecycle.
    - When to use: Raising a question during analysis, answering a question with a source, reviewing open questions, checking gate-rejected questions.
    - Commands: ask (--workstream, --owner), answer (--source), questions list (--owner, --workstream, --json).
    - Rules: Owner taxonomy: self, principal, seller, broker, counsel. Gate-rejected questions appear with [gate] tag. Answers optionally cite a source. Questions are never deleted, only answered.
    - Workflows: "Raise a question" (ask), "Answer with source" (answer), "Review open questions" (questions list).

    **dd_workstreams.md:**
    - Description: Organize deal work into workstreams with tasks.
    - When to use: Creating a new workstream, reviewing workstream status, creating/completing tasks, checking task progress.
    - Commands: workstream new (template names: financial, retention, technical, legal, hr, integration), workstream list (--json), workstream show, task new, task list, task complete.
    - Rules: Workstream names are lowercase-kebab. Each workstream gets a subdirectory with CONTEXT.md and RESEARCH.md. Tasks live in numbered subdirectories. task complete requires SUMMARY.md content. Hand-edits to WORKSTREAMS.md are respected.
    - Workflows: "Set up workstreams" (workstream new for each), "Create and complete a task" (task new, task complete), "Review workstream status" (workstream show).

    **dd_status.md:**
    - Description: Get full deal state summary and generate AI session handoff documents.
    - When to use: Starting the day's work, checking what needs attention, handing off to a fresh AI session, getting a quick deal overview.
    - Commands: status (--verbose, --json), handoff (--since, --everything, --clip, --json).
    - Rules: Status is read-only, never modifies state. Handoff output is a paste buffer. Handoff default window is 14 days. --everything dumps complete state. --clip copies to clipboard.
    - Workflows: "Morning check-in" (status), "Hand off to new AI session" (handoff --clip), "Deep dive on everything" (status --verbose).

    All skill files should be practical and concise. No boilerplate, no filler. Write like a reference card, not a tutorial. Each file should be approximately 60-100 lines.

    @click.command("install")
    @click.option("--claude-code", is_flag=True, default=False, help="Install to Claude Code skills directory")
    @click.option("--antigravity", is_flag=True, default=False, help="Install to Antigravity skills directory")
    @click.option("--path", "custom_path", default=None, type=click.Path(), help="Custom install directory")
    @click.option("--uninstall", is_flag=True, default=False, help="Remove installed skill files")
    @click.option("--json", "json_mode", is_flag=True, default=False, help="Output as JSON")
    def install_cmd(claude_code, antigravity, custom_path, uninstall, json_mode):
    ```

    **Resolve target directory:**
    - --claude-code: `Path.home() / ".claude" / "skills"`
    - --antigravity: `Path.home() / ".agents" / "skills"`
    - --path: `Path(custom_path)`
    - If none specified: raise click.ClickException("Specify --claude-code, --antigravity, or --path")
    - If multiple specified: use --path > --claude-code > --antigravity priority (or error; simplest is to just pick the first one specified)
    - If target directory does not exist: raise click.ClickException(f"Directory does not exist: {target_dir}")

    **Install mode (default, not --uninstall):**
    1. Resolve CLI binary path: `diligent_path = shutil.which("diligent") or "diligent"`. If shutil.which returns None, print warning "Could not find diligent binary in PATH; using 'diligent' as default".
    2. Read each skill template from `SKILLS_DIR / "dd_*.md"` (use the SKILLS_DIR from diligent.skills).
    3. For each template: replace `{{DILIGENT_PATH}}` with the resolved diligent_path using string replace (not string.Template -- the token uses double braces which conflict with Template syntax).
    4. Write parameterized content to `target_dir / filename`.
    5. Print confirmation: `"Installed {count} skill files to {target_dir}"` or JSON equivalent.

    **Uninstall mode (--uninstall):**
    1. Glob `target_dir / "dd_*.md"` to find installed files.
    2. Remove each file.
    3. If no files found: print `"No skill files found in {target_dir}"`.
    4. If files removed: print `"Removed {count} skill files from {target_dir}"`.
