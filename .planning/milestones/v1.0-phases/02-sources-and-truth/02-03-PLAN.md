  <action>
    1. Create helpers/numeric.py with two functions:
       - `try_parse_numeric(value: str) -> float | None`: Strip `$`, `,`, `%`, whitespace using compiled regex `re.compile(r"[\$,\%\s]")`. Return float() or None on ValueError.
       - `compute_gate_result(old_value: str, new_value: str, is_anchor: bool, tolerance_pct: float) -> dict | None`: Implements the locked gate comparison logic exactly as specified in CONTEXT.md. Returns None if no gate fires (no-op, within tolerance). Returns dict with keys: fired (bool), delta_str (str|None), verdict (str) when gate fires.

    2. Create commands/truth_cmd.py with:
       - `truth_cmd` Click group ("truth") with docstring "Manage validated facts."
       - `_next_question_id(questions: QuestionsFile) -> str` helper: derive next Q-{NNN} from max existing question ID.
       - `truth_set` subcommand:
         * Arguments: KEY (required), VALUE (required)
         * Options: --source (required), --workstream (default ""), --anchor (flag), --no-anchor (flag), --confirm (flag), --computed-by (string), --notes (string), --json (flag)
         * Flow:
           a. Find .diligence/ dir. Read config (for deal_code, anchor_tolerance_pct). Read TRUTH.md.
           b. If key exists in truth.facts and value is bytewise equal: exit 0 with "No change" message. (No-op fast path.)
           c. If key exists: determine anchor status (existing anchor field, or --anchor flag, or --no-anchor). Call compute_gate_result. If gate fires and --confirm not set: print compact discrepancy, write rejection to QUESTIONS.md (read current, append new QuestionEntry, write back), exit 2. If gate fires and --confirm set: proceed.
           d. If key exists: push current value to supersedes chain (SupersededValue with current value, source, date).
           e. Determine anchor: if --anchor passed, anchor=True. If --no-anchor passed, anchor=False. Otherwise, preserve existing anchor state (or False for new facts).
           f. Build new FactEntry. Set in truth.facts[key]. Write TRUTH.md. Print confirmation.
           g. --json output: {"status": "created"|"updated"|"discrepancy", "key": ..., "value": ..., "source": ...}

    3. Compact discrepancy output format (locked from CONTEXT.md):
       ```
         key:       {key}
         current:   {old_value} (source: {old_source}, date: {old_date})
         proposed:  {new_value} (source: {new_source})
         delta:     {delta_str}          # only if numeric delta exists
         verdict:   {verdict}. Re-run with --confirm to accept.
       ```

    4. Gate rejection to QUESTIONS.md: create QuestionEntry with:
       - id: next Q-{NNN}
       - question: human-readable discrepancy text (e.g., "{key} changed from {old} to {new} ({delta}). Which value is correct?")
       - workstream: from the fact's workstream
       - owner: "self" (default; analyst needs to decide)
       - status: "open"
       - date_raised: today ISO
       - context: {"type": "gate_rejection", "key": key, "old_value": old, "new_value": new, "old_source": old_source, "new_source": new_source, "delta": delta_str}
  </action>
