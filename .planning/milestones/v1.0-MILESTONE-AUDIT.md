### New Integration Issues (from post-Phase 6 re-audit)

**INT-AUDIT-01 (HIGH):** `doctor.py`, `config_cmd.py`, and `migrate_cmd.py` use `Path.cwd() / ".diligence"` hardcoded. No parent directory walk, no `DILIGENT_CWD` support. Every other command module (10 total) uses `_find_diligence_dir` with parent walk. Running these 3 commands from a deal subdirectory fails.
- Affected: INIT-03, INIT-04
- Note: These 3 commands are typically run from the deal root. This is a consistency issue, not a missing feature.

**INT-AUDIT-02 (LOW):** `truth_cmd.py` uses `ctx.exit(0)` then `return` on the no-op path (line ~148). In Click, `ctx.exit()` does not raise immediately inside a command body. The `return` on the next line is what actually stops execution. Fragile pattern but not currently broken.
- Affected: TRUTH-01

**INT-AUDIT-03 (MEDIUM):** `workstream show` computes stale artifact count using simplified value-changed-only check (`fact.date > a.last_refreshed`). Does not check the source-superseded trigger. Will undercount staleness compared to `artifact list`, `reconcile`, and `status` which all use the full `compute_staleness` engine.
- Affected: WS-03
