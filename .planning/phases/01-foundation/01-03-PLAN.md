  <action>
    1. **diligent/commands/doctor.py**: Click command `doctor`.

       **Flags:** `--json` (structured output), `--strict` (warnings as errors).

       **Three validation layers:**

       Layer 1 -- Existence: Check all 6 files exist in .diligence/. Missing file = ERROR finding with fix: "Run `diligent init` to create missing files."

       Layer 2 -- Parse: For each existing file, attempt to read via its reader module (read_truth, read_deal, etc.). Parse failure = ERROR finding with file path, error message, fix: "Check file syntax. Compare against a known-good file."

       Layer 3 -- Cross-references:
       - Facts in TRUTH.md reference workstreams that exist in WORKSTREAMS.md. Missing workstream = WARNING.
       - Source IDs in TRUTH.md facts follow {DEAL_CODE}-NNN format matching config.json deal_code. Mismatch = WARNING.
       - config.json schema_version is 1. Different = WARNING with fix suggestion.
       - config.json deal_code matches DEAL.md deal_code. Mismatch = ERROR.

       **Finding format:** `{"severity": "ERROR|WARNING|INFO", "file": ".diligence/TRUTH.md", "location": "## annual_revenue", "description": "Fact references unknown workstream 'financal'", "fix": "Check spelling. Valid workstreams: financial, legal, ..."}`

       **Output:** Use output_findings from helpers/formatting.py. Plain text default per CONTEXT.md doctor spec. Summary line at bottom.

       **Exit code:** 0 if no errors (and no warnings in --strict mode). 1 if errors (or warnings in --strict mode).

    2. **diligent/commands/config_cmd.py**: Click group `config_cmd` (registered as "config") with two subcommands:
       - `get`: `diligent config get <key>` -- reads config.json, outputs value. Supports --json. Exits non-zero if key not found.
       - `set`: `diligent config set <key> <value>` -- updates config.json via read_config/write_config. Type coercion: if value looks like int, convert. If looks like float, convert. Otherwise string. Supports --json (outputs new value).

    3. **tests/test_doctor.py**: Create .diligence/ via init (non-interactive), then run doctor. Test clean state. Then corrupt files (delete one, break YAML, add bad cross-ref) and verify findings. Test --json format. Test --strict behavior.

Assert completes in under 200ms. Use average of 3 runs to smooth variance. Mark test with `@pytest.mark.slow` so it can be skipped in fast runs.

    6. **tests/test_json_output.py**: For each command (init, doctor, config get), run with --json flag and verify output is valid JSON. Verify no non-JSON text in stdout when --json is used.
